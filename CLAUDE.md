# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

**AIPREMNA.SK** — Slovak-language educational platform that guides teachers through using AI as a co-pilot. UI and all user-facing content are in Slovak; only `ai_docs/` and `dizajn_stranky/` are internal English-only docs. No user accounts; all state (progress, bookmarks, history) lives in `localStorage`.

The original codebase in this branch was generated by Gemini CLI. When making changes, prefer fixing what's there over rewriting.

## Commands

```bash
pnpm install            # install (pnpm is the package manager — note pnpm-workspace.yaml exists)
pnpm dev                # dev server (see "Dev port" below)
pnpm build              # production build (.output/)
pnpm generate           # static site generation
pnpm preview            # preview built site
```

No lint, typecheck, or test scripts are wired up — `package.json` only has Nuxt's defaults.

### Dev port

Pin dev to `127.0.0.1:3030`, not `localhost:3000`. The user has another Nuxt project bound to `*:3000` (IPv4) that hijacks Chromium's `localhost`. Use `pnpm dev --host 127.0.0.1 --port 3030` when verifying in a browser.

## Architecture

### Three-tier roadmap, sourced from `content/`

The whole site is a static roadmap over Markdown in `content/`, served by `@nuxt/content` v3 (`queryCollection('content')`):

1. **`initial_info/`** — Tier 1, "Základy a Príprava" (foundations: ethics, GDPR, prompting)
2. **`jednotlive_usecases/`** — Tier 2, "Prípady použitia" (use cases grouped by category folder)
3. **`jednotlive_tools/`** — Tier 3, "Konkrétne nástroje" (specific tool guides)

Folder names = category labels. Files = leaf nodes (clickable cards). Each guide ends with a `## ✅ How to know you have understood / Ako zistíte` section whose checklist drives per-card progress tracking.

### Bilingual content + the SK filter

Most guides exist as **SK + EN pairs in the same folder**. There are two naming conventions (see `ai_docs/naming_conventions.md`):

- **General resources** (`initial_info`, `jednotlive_usecases`): different translated filenames, no suffix. E.g. `pisanie_emailu.md` (SK) and `writing_email.md` (EN) live side by side.
- **Tool guides** (`jednotlive_tools/`): same base name, SK gets `_sk` suffix. E.g. `chatgpt.md` (EN) and `chatgpt_sk.md` (SK).

The site shows **only Slovak**. The filter logic is duplicated in two places — keep them in sync:
- `composables/useRoadmap.ts`
- `components/roadmap/ScrollRoadmap.vue`

Both:
1. Keep anything ending in `_sk`.
2. Drop the un-suffixed version when an `_sk` partner exists.
3. Drop a hardcoded list of known EN-only filenames whose SK companion has a different name.
4. Drop `links` files (sidecar YAML for external links — see below).

When adding a new EN+SK pair under `initial_info/` or `jednotlive_usecases/`, the EN filename must be added to the `englishFiles` array in **both** files.

### Card modal flow

The homepage (`pages/index.vue`) renders `RoadmapScrollRoadmap`. Clicking a card emits `node-click` with the guide object; the page sets `?card=<path>` in the URL. `GuideLearningCard` opens whenever `route.query.card` is set, on any page (`index`, `bookmarks`, `recent`).

`LearningCard.vue` fetches the guide via `useAsyncData`, then post-processes the parsed Minimark AST: it walks `body.value` / `body.children`, detects the "verification" heading (matched permissively on `✅`, "ako zisti", "ako vedie", "overeni", "pochopi", "mastered"), and tags each `<li>` under that section with `checklistIndex` + `isInsideVerification`. Those props are read by `components/content/ProseLi.vue`, which `inject`s `checklistState` / `toggleItem` provided by the modal and renders the item as an interactive checkbox.

If you change the AST traversal: progress tracking, the sidebar `ProgressTracker`, and the inline checkboxes all break together.

### External links sidecar

A guide can have a `links` file in its directory (e.g. `links.md` or `links.yml`). `useContent.fetchGuide` and `LearningCard.vue`'s loader both look for `${dir}/links` and attach `linksData.links` to the guide as `externalLinks`. The roadmap filters exclude any path ending in `/links` so these never appear as cards.

### localStorage composables

All persistence is client-only and keyed by guide path. Each composable guards `typeof window !== 'undefined'`:

| Composable | Key pattern | Shape |
| --- | --- | --- |
| `useProgress(path)` | `progress-${path}` | `Record<number, boolean>` keyed by checklist index |
| `useBookmarks()` | `bookmarks` | `string[]` of paths |
| `useLastUsed()` | `last-used` | `LastUsedItem[]` capped at 20, added after 10s open |
| `useSearch()` | none — in-memory FlexSearch index of title+description+body |

`ScrollRoadmap.getProgress(path)` currently fakes the denominator as `5` because it doesn't have access to the actual checklist length without loading every guide. Treat per-card % on the roadmap as approximate.

### Styling

Tailwind with a custom `brand` palette in `tailwind.config.js` (`primary` purple `#8175fb`, `secondary` blue `#3d72e5`, `accent` light blue `#7dbcf9`, `background` `#f8f2f2`, `dark` `#140c0b`). Fonts: `Krona One` (headings) + `Ubuntu` (body) via `@nuxtjs/google-fonts`.

## Content authoring

When adding a new guide, follow the templates in `ai_docs/`:
- `ai_docs/usecase_guide_standard.md` for `jednotlive_usecases/`
- `ai_docs/tool_guide_standard.md` for `jednotlive_tools/`
- `ai_docs/naming_conventions.md` for filenames (snake_case, no accents)

The `✅ How to know / Ako zistíte` section is **required** for the progress checklist to work. Bullet items in that section become the trackable checklist.

## Things that look broken but aren't (yet)

- `components/roadmap/TreeRoadmap.vue` exists but is unused — the homepage uses `ScrollRoadmap`. Vue Flow is in `package.json` but not actually imported anywhere in the current render path.
- `pages/cheatsheet.vue` template calls `window.print()` directly — works in SPA mode but will throw during SSR/SSG. The button is `hidden md:flex`, so it usually isn't reached during prerender, but a real fix is to wrap it in a method.
- The mobile menu button in `components/common/Header.vue` is a visual stub with no click handler.
- `Header.vue` calls `useSearch()` and `useBookmarks()` etc. without explicit imports — relies on Nuxt auto-imports from `composables/`. Don't add explicit imports for these; Nuxt will warn about shadowing.
